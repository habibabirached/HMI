⚙️ Layer 2: Python Backend
FastAPI Application Server • Python 3.10+ • Async/Await
🔍 Key Relationship: FastAPI endpoints (app.py) call Business Logic modules (database.py, models.py, schemas.py). Business Logic calls SQLAlchemy which calls PostgreSQL. Backend also orchestrates FlexSim for PSCAD integration.
POST /api/save → Saves configuration to PostgreSQL via SQLAlchemy
GET /api/load/{id} → Queries PostgreSQL, returns configuration JSON
GET /api/configs → Lists all configurations from database
DELETE /api/configs/{id} → Deletes from PostgreSQL + disk backup
POST /api/simulate → Calls FlexSim → PSCAD → Returns results

🔗 FlexSim Automation Tool
Python Bridge Between Backend and PSCAD (Developed by Megan)
🔍 Key Relationship: FlexSim is a Python module that our backend imports and calls. FlexSim wraps PSCAD's Python API, handles CSV configuration, manages simulations, and returns pickle files. This allows our backend to control PSCAD without knowing PSCAD internals.
📝 Component Parameterization
Reads CSV configs → Sets PSCAD parameters
⏱️ Event Scheduling
Configures faults, breaker operations, load changes
▶️ Simulation Execution
Calls PSCAD API → Monitors progress → Collects results
📊 Post-Processing
FFT analysis → RMS calculations → Pickle serialization
🔬 PSCAD Python API
Backend/FlexSim sends commands to PSCAD (load model, set params, run simulation)
PSCAD executes simulation and returns results (CSV, Pickle, or direct data)
🔬
🔬 Layer 4: PSCAD Simulation Engine
Electromagnetic Transient Simulation • Power System Physics Engine
🔍 Key Relationship: PSCAD is an external simulation software with a Python API. It doesn't directly know about our HMI. FlexSim translates our high-level requests into PSCAD commands. PSCAD performs the physics simulation and returns raw results.
