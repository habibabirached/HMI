Software architecture — data & presentation (graph view)

Multi-path topology: simulation → ingestion / Docker-hosted services → AWS (Lambda & edge) → three modalities (desktop, iPad, control room). React, Redux, and Plotly are first-class. NUI as a service augments every client.

simulation & ETL CI/CD → Docker AWS / edge Lambda / async SQL / ORM API → clients NUI (dashed)
Non-linear story: Physical simulation (RTDS, PSCAD) feeds FlexSIM (Python). Writes land in PostgreSQL inside a Docker boundary together with FastAPI. Jenkins / CI/CD builds and promotes images into that boundary. AWS (API Gateway, Lambda, S3) extends the platform for burst workloads, webhooks, and edge-style integrations — Lambdas can call back into the API or enrich NUI. Three intentional modalities—desktop, iPad, control room—all use React and Redux (not optional) with Plotly for multi-dimensional visualization. NUI as a service (voice, gesture, media streams) plugs into each client and can offload heavy perception to Lambda. Diagram is a logical graph; curve routing is for clarity, not literal network hops.
